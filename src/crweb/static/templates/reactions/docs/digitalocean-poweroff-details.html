DigitalOcean: Power Off Droplet

Overview

The DigitalOcean: Power Off Droplet reaction can be used to power off cloud servers (aka Droplets) hosted with DigitalOcean.

Reaction Actions

This reaction will only perform a single reaction, this reaction can be configured to occur on "Failed Monitors" or "Healthy Monitors" but is not designed to be executed on both. In order to setup this reaction to have an action for "Healthy Monitors" and a different action for "Failed Monitors" you can create two reactions and attach them to the same Monitor.

Failed Monitors

If this reaction's Call On field is set to "Failed Monitors".

When this reaction receives a "Failed Monitor" the reaction will send an API request to DigitalOcean requesting a power off of the droplet specified.

If this reaction's Call On field is set to "Healthy Monitors".

This reaction will perform no tasks when it receives a "Failed Monitor"

Healthy Monitors

If this reaction's Call On field is set to "Healthy Monitors".

When this reaction receives a "Healthy Monitor" the reaction will send an API request to DigitalOcean requesting a power off of the droplet specified.

If this reaction's Call On field is set to "Failed Monitors".

This reaction will perform no tasks when it receives a "Healthy Monitor"

Reaction Form Details

API Key

The API Key field should contain your API key in the new version of DigitalOcean's API this is called a Personal Access Token.

Droplet ID#

The Droplet ID# field should contain the ID number of the droplet you wish to power off, this can be found in the URL of the droplet details.
