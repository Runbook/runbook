HTTP GET: Keyword

Overview

The HTTP GET: Keyword monitor allows you to perform an HTTP or HTTPS GET request on a specific page and search for a keyword. The keyword can be both a string or a regular expression, this can be used to validate that your app is serving content correctly or search the page for errors.

Monitor Details

This monitor will perform an HTTP GET on the specified url and page; the returned text will be searched for the keyword specified.

URL

This field is used to target the HTTP GET request. This can be a domain or an IP address; however the entry must be in URL format such as https://10.0.0.1/index.html.

Domain to request

The domain field is used to specify the Host header of the request. This is useful for requests that address an IP address rather than the domain itself.

Using the URL and Domain to request

If you wanted to monitor the website at http://example.com/status.html and example.com resolved to 10.0.0.1 and 10.0.0.2 you could monitor each server by setting up two monitors. Each monitor would address the servers IP directly and use the Domain to request field to specify which domain to validate.

Example Monitor #1:
URL: http://10.0.0.1/status.html
Domain to request: example.com

Example Monitor #2:
URL: http://10.0.0.2/status.html
Domain to request: example.com

This would allow you to monitor each server independently, and setup unique reactions based on each server.

Keyword

The Keyword field is the string or regular expression the monitor should search for. This performs a basic search amongst all text within a page, it is important that you ensure the search text or pattern is unique to avoid false positives.

Is Keyword a Regular Expression

This selection field allows you to specify if the keyword provided is in regular expression format or not. If you are not using a regular expression you should set this to false.

Mark Healthy if keyword is present

This selection field allows you to specify whether the monitor is healthy when the keyword is present, or absent. You can specify this as false if the keyword was searching for unwanted errors such as database connection errors or any other error that would be presented as page text.
