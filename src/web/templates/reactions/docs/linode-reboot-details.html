Linode: Reboot Server

Overview

The Linode Reboot Server reaction can be used to Reboot a Linode server (aka linodes).

In most circumstances Rebooting a server is a last ditch effort. However when using automation to resolve issues it is a viable solution to issues that are not resolved by other reactions. For example, if you have a monitor checking the status of Apache. This first reaction of that monitor for failure should be to restart Apache, a reasonable trigger time for this is 3 with a frequency of 300 seconds. If after 40 false monitors Apache is still not responding, it is not unreasonable to Reboot the server rather than wake someone up.


Reaction Actions

This reaction will only perform a single reaction, this reaction can be configured to occur on "False Monitors" or "True Monitors" but is not designed to be executed on both. In order to setup this reaction to have an action for "True Monitors" and a different action for "False Monitors" you can create two reactions and attach them to the same Monitor.


False Monitors

If this reaction's Call On field is set to "False Monitors".

When this reaction receives a "False Monitor" the reaction will send an API request to Linode requesting a Reboot of the specified linode server.

If this reaction's Call On field is set to "True Monitors".

This reaction will perform no tasks when it receives a "False Monitor"


True Monitors

If this reaction's Call On field is set to "True Monitors".

When this reaction receives a "True Monitor" the reaction will send an API request to Linode requesting a Reboot of the specified linode server.

If this reaction's Call On field is set to "False Monitors".

This reaction will perform no tasks when it receives a "True Monitor"


Reaction Form Details

API Key

The API Key field should contain your API key.


Linode ID#

The Linode ID# field should contain the ID number of the server you wish to reboot.
