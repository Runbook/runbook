General Reaction Options

Name

The name field is a required field that must be unique within your account. The name field should contain Alphanumeric input and does allow for special characters.

Trigger

The Trigger field allows you to specify how many failures must occur before a specific reaction is executed.

As an example, one could setup an Email Notification reaction to occur after the 2nd instance of a failure by setting the reactions trigger to 2.

It is important to note that Triggers are for both True and False monitors. When a monitor changes from False to True a counter is zeroed; any true reactions will not be executed until the monitors counter has reached the reactions defined trigger.

Frequency

The Frequency field allows you to specify in seconds how often a reaction is executed. This can be used to reduce or increase the frequency a reaction is executed. For example, if a monitor runs every 5 minutes and the frequency of the reaction is set to 600 seconds the reaction will only execute every 10 minutes.

Call On

The "Call On" field is used for reactions that do not necessarily have a intuitive action for both "False" and "True" monitors. For example, the Email Notification reaction has both a "False" step and a "True" step. That reaction will notify the specified email address of a failure, and when the monitor returns true it will notify the email address of the return to true status.

While developing new reactions we have found that some tasks however, do not have secondary action. These reactions should only be called once, either during a "False" monitor or during a "True" monitor. If you wanted to use the same reaction but with different values for "False" and "True" you can accomplish that by creating two reactions. The first reaction would have a "Call On" value of "False Monitors" and execute the required steps during failures. The second reaction would have a "Call On" value of "True Monitors" and execute the required steps during true instances.
